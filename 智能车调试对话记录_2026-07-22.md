## 2. 最初问题：最后一行全白导致卡帧

### 现象

摄像头拍到的图像最后一行如果是全白像素点，屏幕图像会卡在那一帧，车也一直按那一帧找边界。

### 分析结论

主要不是摄像头卡死，而是图像处理逻辑可能进入越界或异常循环。

重点问题在：

```c
find_base_point();
find_boundary();
```

当最后一行全白时，`find_base_point()` 容易得到：

```c
base_point_left = 2;
base_point_right = MT9V03X_W - 3;
```

随后 `find_boundary()` 里有多个 `uint8 j--` 递减循环，例如：

```c
for(uint8 j=left_point; j>left_point-left_search_left_range; j--)
```

当 `left_point = 2` 时，`uint8` 会下溢：

```text
2, 1, 0, 255, 254, ...
```

并且访问：

```c
twovalues_image[i][j-1]
twovalues_image[i][j-2]
```

会越界。

### 重要建议

所有图像列递减搜索建议用：

```c
int16 j;
```

并且访问 `j - 2` 前保证：

```c
j >= 2
```

访问 `j + 2` 前保证：

```c
j <= MT9V03X_W - 3
```

## 3. 右转环岛右边线拟合中间圆的问题

### 现象

左转可以找到左边线，但右转时右边线会拟合环岛中间的圆，表现为右边线沿圆岛边缘向上延伸。

### 核心原因

右边界判断条件类似：

```c
255, 255, 0
```

也就是白到黑的跳变。

普通赛道右边界和环岛中间圆的左边缘，在二值图中都可能满足这个特征。

所以右转环岛时，`right_line[]` 很容易变成：

```text
环岛内圆边界
```

而不再是普通赛道右边界。

### 关键认知

`right_line[]` 拟合中间圆并不是完全没用，但语义已经变化：

```text
普通道路：right_line = 赛道右边界
右环岛中：right_line = 环岛内圆边界
```

所以不能继续直接使用：

```c
mid_line[i] = (left_line[i] + right_line[i]) / 2;
```

环岛状态下应该根据状态机单独补线。

## 4. 环岛状态机当前设计思路

### 主循环位置

环岛状态机应放在：

```text

STEP_STEER 之前
```

当前主循环结构大致为：

```c
case STEP_BOUNDARY:
  find_boundary();
  ...冲出赛道保护...
  step = STEP_RING;
  break;

case STEP_RING:
  ring_state_process();
  step = STEP_STEER;
  break;
```

## 5. 右环岛状态机演进记录

### 当前状态枚举

```c
enum {

    ring_state_wait_right_near_lost,
    ring_state_left_straight,
```

### 当前右环岛状态流程

```text

  ↓ 检测到入口特征

  ↓ 连续确认入口特征

ring_state_in
  ↓ 环内处理
ring_state_confirmed
  ↓ 出环确认
ring_state_idle
```

## 8. 信左边界行驶阶段

进入 `ring_state_left_straight` 后，才开始信任左边界行驶。

后来发现不能简单：

```c
mid_line[i] = left_line[i] + 固定值;
```

因为图像存在透视关系：

```text
远处赛道窄，偏移应小
近处赛道宽，偏移应大
```

所以现在改成按行偏移数组：

```c
ring_left_straight_offset_table[i]
```

计算方式是线性插值：

```text
search_end_line 附近使用 FAR 偏移
MT9V03X_H - 1 附近使用 NEAR 偏移
```

相关参数：

```c
#define RING_LEFT_STRAIGHT_OFFSET_FAR  35
#define RING_LEFT_STRAIGHT_OFFSET_NEAR 60
```

## 12. 转向 PD 优化

### 原始转向

原来使用单行：

```c
int16 error = mid_line[110] - 94;
int16 error_far = mid_line[30] - 94;
float steering = Kp_steer * error + Kd_steer * (error - error_far);
```

问题：

- 单行误差容易抖
- `error - error_far` 不是时间 D，而是近远误差差
- `mid_line[30]` 太远，容易受噪声影响

### 当前转向结构

在 `main.c` 里加了多行平均：

```c
get_mid_error_average(start_row, end_row)
```

当前行段：

```c
#define STEER_NEAR_ROW_START  100
#define STEER_NEAR_ROW_END    115
#define STEER_FAR_ROW_START   45
#define STEER_FAR_ROW_END     65
```

当前转向公式：

```c
int16 error_near = get_mid_error_average(STEER_NEAR_ROW_START, STEER_NEAR_ROW_END);
int16 error_far = get_mid_error_average(STEER_FAR_ROW_START, STEER_FAR_ROW_END);
int16 preview_error = error_far - error_near;
int16 d_error = error_near - last_error;

float steering = Kp_steer * error_near
               + Kd_steer_position * preview_error
               + Kd_steer_time * d_error;
```

并加入：

```c
#define STEER_MAX_RATIO 0.85f
#define STEER_MAX_STEP  10.0f
```

即：

- 转向输出限幅
- 转向变化率限制

停车时会清：

```c
last_error = 0;
last_steering = 0;
```

## 13. 关于 Kd_steer_time

`Kd_steer_time` 调大后小车变迟钝，是正常现象。

原因：

```text
时间 D 项本质是阻尼，用来抑制误差变化和摆动
```

它太大时会：

```text
压制转向变化
入弯迟钝
回正变慢
```

而且当前还有：

```c
STEER_MAX_STEP
```

会进一步限制转向变化。

建议调参范围：

```text
Kp_steer: 0.70 ~ 0.90
Kd_steer_position: 0.10 ~ 0.25
Kd_steer_time: 0.005 ~ 0.02
STEER_MAX_STEP: 10 ~ 15
```

想让车更灵敏，优先调：

```text
Kp_steer
Kd_steer_position
```

不要优先增大：

```text
Kd_steer_time
```

## 14. 直道偏离中心线问题

只调 `Kp_steer` 小车仍然走不了直道，通常不是 Kp 本身的问题，而是有固定偏置。

可能原因：

1. 图像中心列 94 不等于真实车体中心
2. 摄像头安装偏
3. 左右轮速度不一致
4. 速度环左右响应不一致
5. `mid_line[]` 被环岛状态机误改
6. 近处行段太靠底部，受白行、车头、曝光影响

建议排查顺序：

1. 临时关闭 `ring_state_process()` 测直道
2. 低速测试，比如 `run_speed = 80 ~ 100`
3. 校准图像中心列
4. 检查左右轮 `real_speedl` / `real_speedr`
5. 再调 Kp

## 16. 按键启动逻辑

### 需求

K4 不要硬把速度设置成固定值，而是使用菜单中可调速度。

### 当前做法

在 `motor.c` 中：

```c
int base_speed = 0;       // 当前运行速度，0 表示停车
int run_base_speed = 100; // 菜单可调启动/巡线速度
```

在 `motor.h` 中：

```c
extern int base_speed;
extern int run_base_speed;
```

在菜单中：

```c
dynamic_create_menu_txt(xunxian_folder, "run_speed", &run_base_speed, int32_box);
```

K4 双击逻辑：

```c
void key_4_double(void) {
  motor_pid_reset();
  target_speedl = 0.0f;
  target_speedr = 0.0f;

  if (base_speed == 0) {
    base_speed = run_base_speed;
  } else {
    base_speed = 0;
  }
}
```

运行中允许菜单实时调整速度：

```c
if (base_speed > 0) {
  base_speed = run_base_speed;
}
```

### 使用方式

```text
开机不跑
菜单 xunxian -> run_speed 调速度
双击 K4 启动
再双击 K4 停止
```
